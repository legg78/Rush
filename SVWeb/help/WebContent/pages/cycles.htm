Cycles

Cycles are used to set the frequency. Cycles can be auxiliary (used in fees and limits) and independent (for example, the cycle of calculating the card life cycle).

The following parameters are matter on the calculation:

•Cycle length type, that is, a cycle unit (year, month, week, day, hour)

•Cycle length, that is, the number of units from the reference date

For cycle fine-tuning the user should use shifts. Shift has a type (algorithm) by which it works:

•Shift by period

•Shift to certain workday, if the calculated day is a holiday

•Shift to exact day of week/month

Shift also has a sign that defines the direction to produce the offset (forward or backward). As a cycle, shift has length type and length. On different algorithms these parameters are interpreted differently:

•For shift to the working day, these parameters are ignored

•For shift on the day of the week/month, the shift length is the number of the day of the week (1 - Monday, 2 - Tuesday, etc.) / month (1 - January, 2 - February, etc.)

The number of shifts for a cycle is not limited. Each shift has an order number which should be used to cycle.

For example, the task is to set a cycle day as the last Wednesday of the month for a working day and the nearest working day if the Wednesday is nonworking day. In this case, set up the cycle from the beginning of the month, the cycle length type is a month, length - one. There are three shifts:

1.Truncation type — month.

2.Shift back one day — get the last day of the month.

3.Find the day of the week (Wednesday) back from the calculated date.

4.Shift forward to the next working day, if the previous shift result is nonworking day.

Cycles are created when values are specified for service terms of the cycle type or fee/limit type with a cycle.

On applying a cycle to the exact object, the number of the current period, start date and end date is fixed. The period end date is the start date of the next cycle period.

Any cycle must be initiated. Cycle initialization is based on events: the event must be registered with the system; a rule set with the cycle initialization procedure must be configured and assigned to the event.
