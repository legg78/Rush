Product for virtual card with restricted available balance calculation

In this example of card product configuration, you are required to configure a product that enables Rouble accounts to be opened for individual bank customers and for virtual cards to be issued to them.

The available balance applied to each card is calculated as follows:

1.The current account balance is determined.

2.On the basis of the limit scheme assigned to the card, a list of limits participating in the available balance is defined.

3.The unspent balances of the defined limits are determined (if the limit and account currencies are different, the limit balance is converted to the account balance currency).

4.The lowest value among the unspent balances in these limits and the current account balance is determined. This value will be the card balance.

The steps to configure this example are listed below.

1.Repeat configuration steps 1–7 described in the Product for virtual card issuing topic.

2.Click Configuration >Arrays > Array types.

3.Configure the array type limits to determine the available balance and add the required limit types to it.

4.Specify the service terms for the card product:

a.Click Issuing > Configuration > Products, select the product and click Service terms.

b.In the Service terms tab, click Limits and configure the List of limits for limitation of available balance parameter.

For the parameter value, specify Limits to determine the available balance configured earlier.

5.Configure the card processing rules:

a.Click Operational rules > Processing > Procedures, and configure the Get card balance procedure parameters.

b.Click Operational rules > Processing > Rules sets, and configure a rule for the Get card balance procedure.

c.Click Operational rules > Processing > Templates, and configure templates for the balance inquiry transactions using the Get card balance rule configured in step b.

Parameter

Description

Amount name

The name of the amount used for calculation.

Lock needed

Determines whether the account is locked on the balance inquiry.

Effective date

The settlement date type (bank date or system date).

Party Type

The participant type. Set the Party type to Issuer for this procedure. The necessary account and the card are defined according to the issuer.

Note: For the issuing institution, specify the institution using the configured card product.
