Application stages

An application creates or modifies the data of a customer object (for example, cards, accounts, customer, or contract) within a particular contract.

In the SmartVista web interface, any application is created in three stages:

1.The user specifies the objects that should be created.

2.The user configures the parameters of the objects.

3.The application is processed.

Stage 1

In the first stage, the user defines the application parameters and necessary objects for creating or editing. The following parameters can be defined:

•The institution and its agent under which the application is to be registered.

•The product that describes the service terms for applicable to customer servicing.

•The customer block: a previously registered customer number or a new customer type.

•The contract block: a previously registered contract number or a new contract type.

•The application processing flow.

•The objects to be created or edited.

Stage 2

In the second stage, the user specifies an object’s data via the application page. The page consists of sections, each of which contains a set of text boxes and lists that define the object parameters. For example, the Cardholder section contains a set of parameters for the cardholder identification.

Some sections are connected hierarchically. Thus, the Card section always has a Cardholder child block, which includes the Contact, Person, etc.

Some parameters may not be editable. Their values are conditional upon the application business process. In some sections you can add additional parameters (such as an additional address, contact, service terms that are defined at the object level).

Card number, account number, cardholder name, and other names can be manually entered or generated automatically based on the naming rules. Manually entered values take precedence over generated ones. If a particular object has no predefined rule for automatic naming and the name or number of the object is not explicitly entered, then the processing will generate an error.

Note: These fields are not marked with asterisk, as it is assumed that the automatic name generation rule was set and manual entry of the name is required in exceptional cases.

 

Stage 3

In the third stage, the application is processed by the dedicated SVBO process. For more information, see Application processing. As a result of processing, the objects specified in the application are created in the system.
