Credit processing configuration

The credit service uses an event-driven model. Events are generated by external modules (for example, transactions, reversals, disputes) and the inside of the module (cycles). Each event is bound a list of actions that must be performed in connection with a particular event. Credit processing on the first stage is verification of all cycles that are configured in the credit module and generation of the event if the date is less than the current cycle date. Then, in the second stage, there is a complete sample of all events generated by both cycles and external modules, and their consistent processing. All events are processed in the order they occurred in the system. The end cycle event occurrence date is the end of the cycle but not the date when the end of the cycle was detected.

To display bunches of transactions of credit period events:

1.Click the Operational rules > Accounting > Credit life cycle menu items.

 

2.Specify the search parameters. Leave the fields blank to list all bunches for credit period events.

Parameter name

Description

Institution

Institution associated with the account.

Balance type

Target balance that is used in transactions.

Event type

Event that causes the target transaction.

3.Click Search.

The search result panel will display bunches by the certain balance type that should be processed when the certain event of the credit period is occurred. The panel has controls to customize the processing of the credit period (Add, Edit, or Delete).

The Transaction templates details panel tab displays a set of the selected bunch transactions that will take place upon the occurrence of the relevant event. The tab is divided into two parts: the left side displays a list of templates for the selected event, and on the right, the selected transaction template parameters are displayed. The panel has controls to view the full information about a transaction, to add a template, to change the bunch, to remove a transaction template from the bunch.
