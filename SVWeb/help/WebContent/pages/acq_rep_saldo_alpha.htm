Pay roll

Report name — Pay-roll

Report description — Pay roll on given terminals over clearing date specified period (includes DMS and SMS operations).

The report is generated on all payment systems (China UnionPay, MasterCard, Visa, Diners Club) as of SVBO clearing date by the results of posting file processing in the system.

Report header — Pay roll on operations on firm terminals merchant id (merchant name).

Report building parameters

Parameter name

Description

Institution

Identifier (numerical code) of acquirer institution.

Terminal type

Terminal type:

•ATM

•POS (the default value)

•IMP – imprinter

•EPOS

Start Date

Clearing period start date. By default, the start date is the current date. The report includes the transactions processed during the specified period.

End Date

Clearing period end date. The report includes the transactions processed during the specified period.

Time Cut-Off

Time of SMS operation cut off in HH.MI.SS. format. The default value is 13.00.00

Merchant ID

Identifier of the merchant.

Terminal ID

Terminal registration number.

Report fields

Name in the report

Description

Organization

Merchant ID and name. The value meets the report parameter set by the user.

Terminal

Terminal identifier. The value meets the report parameter set by the user.

Currency

Terminal number in the transaction.

Date/Time

Transaction date and time.

Auth. Code

Authorization code.

Card number

Card number in the transaction.

Payment Amount

Payment transaction amount.

Cardholder Fee

Total sum of the cardholder fee.

Transaction Sum

Transaction total amount.

Card Program

Calculated field.

SMS/DMS

Message system (Single Message System of Dual Dual System).

FE Utrnno

Transaction unique number in FE.

Grouping — by merchants, terminals, transaction currency.

Grouping

Grouping header

By merchants

Organization merchant name (merchant id)

By terminals

Terminal terminal id.

On currency

Currency currency code.

Sorting — by date.

Totals — by groupings by terminals

Header in the report

Description

Date/time Total Under The Line Cutt-off SMS

Total sum of Advance amount, Cardholder fee, and Transaction sum fields on the terminal in the specified currency for SMS operations (for specified date and by specified time). Thus, the total value is composed of SMS operation sums and fees executed by the time specified in report building parameters. See Note below.

Total Cashing

Total sum of Advance amount, Cardholder fee, and Transaction sum fields of the terminal in specified currency for SMS operations minus corresponding cashing amounts.

Total

Total (as regards payment systems) – Total sum of “ Advance amount, Cardholder fee, and Transaction sum fields on terminal in specified currency for each payment system.

Total Of Terminal terminal id (currency code)

Total sum of Advance amount, Cardholder fee, and Transaction sum fields of terminal in specified currency for SMS operations.

Total Of All Terminals (currency code)

Total sum of Advance amount, Cardholder fee, and Transaction sum fields of all merchant terminals in specified currency.

Note: There are certain specifics regarding generation of the total sum called Date/time Total Under The Line Cutt-off SMS. If a report specifies the time of an SMS operation registering prior to which (on certain date) no transactions occurred, the totals header displays the following:
 
Date ----- Total Under The Line Cutt-off SMS
 
It means that the Total is calculated at the end of the day instead of at the time specified in the report building parameters (because no transactions occurred by this time).
