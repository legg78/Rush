Setting the cycle value

The user sets a cycle value on the level that was specified as the Definition level of the corresponding service term: service, product and object, object.

To set the cycle value:

1.Access the page that corresponds to the service term definition level.

2.Click Search on the search pane.

3.Select the service/product/object (for example, Card, Account, Terminal) to set the service term value.

4.Click the Service terms tab on the details pane.

A warning icon means that the service term cannot be set on this page (level):
 

5.Select a cycle (or a cyclic service term) and click Set.

Note: Depending on the service term parameters set on its creation, other tabs can also be displayed (for example, the Fee or Limit tabs).

6.On the Set service term value page (or on the Cycle tab, if the page contains several tabs), specify the cycle parameters value:

a.Select the radio button that defines the method of setting the value:

•Create new value – setting all parameters of a new value.

•Clone value – duplicate the previously set value, all parameters of the cycle value can be edited.

•Apply value – duplicate the previously set value, only the period of validity can be edited (the Start date and End date combo boxes).

Note. If you selected Clone value or Apply value, select the cycle which value must be duplicated from the Cycle list.

b. Specify the cycle parameter values:

Parameter

Description

Truncation type

The method to choose the cycle start date. For example, on creating a working day cycle, the user can set a calendar day (day). The cycle will start counting from the beginning of the day. On a month rounding the cycle starts counting from the beginning of the month, on a year rounding – from the beginning of the year.

If the parameter is not set then current date, hour, minute, second will be the start date.

Length type

Unit of the cycle length measurement.

Cycle length

Cycle period.

c. To set a cycle shift, click Add and specify the shift parameters.

Shifts are used to fine-tune cycles. For example, a shift ensures that the cycle end date does not fall on the wrong day (for example, a weekend or holiday). You can set several shifts for a cycle. Shifts are executed in the given order; they have a sign that indicates the direction of the shift.

 

Parameter

Description

Shift type

Type of the shift (for example, on period, on a date, on a day of a week, on a working day).

Priority

Order of the shift.

Shift sign

Direction of the shift: forward (+) or back (-).

Shift length

Size of the shift.

Length type

Period type (for shift by a period only).

When choosing a shift on the last day of the month, the Shift sign and Shift length parameters are ignored.

Note: To set/edit the shift select it and click Delete/Edit accordingly.

d. In the Start date and End date combo boxes specify the fee validity. If the end date is not set then it is an indefinite period.

Note: If the scale was set for the service term then the Modifier list is displayed below the period of validity combo boxes. The user can select the previously created condition on which the service term value is applied.

For the description of the Fee and Limits tabs (service terms), see in the corresponding chapters on fees and limits.

7. Click Save.
