Adding a process to a container

1.Click Administration > Processes > Containers.

2.Specify search parameters on the search pane and click Search.

3.Select the container for which a process should be added to.

4.Click the Processes tab.

5.Click Add.

6.Specify the process parameters.

Parameter

Description

Process

The process of the institution needed to add to the container

Execution order

The order of the process running in the container

Error limit

The available rate of errors that defines the process execution as successful. You may use scroll on the window of type in manually. Possible values: from 1 to 100, integer

Track threshold (items)

The number of processed items after which the system begins tracking the number of errors.

Possible values: integers from 1 to 9999.

Parallel

Specifies that process will be executed using multiple threads.

Use custom value

Enable the Degree of parallelization parameter.

Stop on fatal

Process will be stopped if a fatal error occurs.

Description

Full description of the process.

Language

Language of the input text data.

Degree of parallelization

Number of threads to execute the process.

Trace level

The level of tracing for the process:

•Trace off — no events are logged

•Fatal error — only fatal errors (which lead to system stopping) are logged

•Error — all errors that occur during the system processing are logged

•Warning — in addition to errors, warnings of situations that do not themselves cause errors are logged. Warnings provide an indication of issues that should be investigated.

•Info — all successful events

•Debug — system debug events

Debug writing mode

The mode of recording debugging messages into the log:

•LGMDIMDT - Immediate saving

•LGMDNRRR - On error

•LGMDSSPD - Suspended recording

Start trace size

Initial number of debug messages to be written to the container trace log.

Error trace size

Number of debug messages to be written before the error message (if an error occurs).

7.Click Save.
