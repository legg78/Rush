Operations unloading

Process ID and name

10000047 - Operations unloading

Process function

The process uploads data on transactions.

Process parameters

Parameter

Description

Mandatory/Optional

Institution (I_INST_ID)

An identifier of a financial institution which operations should be uploaded. The value is selected from the list. If a definite institution need not be specified, select 9999 = Default/All.

O

Date from (I_START_DATE)

The initial date of the file upload interval. The value is selected from the system calendar.

O

Date to (I_END_DATE)

The final date of the file upload interval. The value is selected from the system calendar.

O

Event type used for upload (I_UPL_OPER_EVENT_TYPE)

A type of an event for which transaction data should be uploaded. The value is selected from the list:

•EVNT0280 - On us operation processing

•EVNT1710 - Operation successfully processed

•EVNT1720 - processing operation failed

O

Terminal type (I_TERMINAL_TYPE)

A type of a terminal device for which transaction data should be uploaded. The value is selected from the list (the TRMT dictionary - terminal types):

•TRMT0001 - Imprinter

•TRMT0002 - ATM

•TRMT0003 - POS

•TRMT0004 - ePOS

•TRMT0005 - Mobile

•TRMT0006 - Internet

•TRMT0007 - Mobile POS

O

Status of the loaded operations (I_LOAD_STATE)

A status of data to be uploaded. The value is selected from the list:

•All

•Already loaded

•Not loaded before

O

Load reversals (I_LOAD_REVERSALS)

An option enabling/disabling upload of data on reversals. The value is selected from the list. Available values:

•0 – No – data on reversals is not uploaded

•Yes - data on reversals is uploaded

O

Load reversed (I_LOAD_REVERSED)

An option enabling/disabling upload of data on reversed transaction. A logical value is selected from the list. Available values:

•0 – No – data on reversed transactions is not uploaded

•1 – Yes - data on reversed transactions is uploaded

O

Which operations will be uploaded (I_LOAD_SUCCESSFULL)

A type of transactions according to a completion status. The value is selected from the list. Available values:

•All - data on all transactions is uploaded

•Only declined - data only on rejected transactions is uploaded

•Only successful - data only on successful transactions is uploaded.

O

Include authorization data (I_INCLUDE_AUTH)

The option enabling/disabling the upload of data on authorizations:

•0 – No – data on authorizations is not uploaded

•1 – Yes - data on authorizations is uploaded.

O

Include clearing data (I_INCLUDE_CLEARING)

The option enabling/disabling the upload of clearing data:

•0 – No – clearing data is not uploaded

•1 – Yes - clearing data is uploaded

O

Masking card (I_MASKING_CARD)

A card masking option. A logical value is selected from the list. Available values:

•0 – No – card numbers in accounting entries are not masked

•1 – Yes – card numbers in accounting entries are masked

O

Files

The process uses outgoing clearing files in XML format.
