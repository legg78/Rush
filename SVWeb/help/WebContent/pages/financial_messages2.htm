China UnionPay financial messages

The Financial Messages window displays financial messages generated when processing China UnionPay authorizations.

1. Select Payment System > China Union Pay > Financial messages.

2. Specify the search parameters in the search pane:

Parameter

Description

Operation ID

The transaction identifier.

Session ID

An identifier of a file load session.

Card mask

The masked card PAN.

Reference number

The card reference number.

Transaction amount

A transaction amount in the transaction currency.

Transaction date

The transaction execution date.

Direction

The financial message direction (incoming or outgoing).

3. Click Search.

The search result pane displays a list of messages according to the specified conditions.

Additional information about a selected message is available in the Details, Cross-border features and IC data tabs under the list of messages.

Details tab

Parameter

Description

Transaction currency code

A three-digit value representing the transaction currency used during authorization

System Trace Number

The System Trace number is used in the event of a dispute.

If the system trace number is not included in the original transaction, the value is assigned by CDRS, which is unique within one day. The value will remain unchanged during transaction cycle

Authorized response Identifier code

An identifier code provided by the card issuer in a response to the authorization.

The value of this field should be equal to the value in field 38 of original online message. For offline purchase of IC card debit/credit application, the default value of this field is filled with space. For offline purchase/refund of IC card E-cash application, this field is filled with the tag 9F74 value (the issuing bank authorization code)

Date of authorization

The date (MMDD) when the authorization for this transaction was received. This date is equal to the value of field 15 Settlement Date in the original authorization transaction message.

For offline purchase and offline refund of UICS debit/credit application and e-cash application, the default value of this field is filled with spaces

Agency Identifier Code

The code value is left-justified, filled with spaces on the right if the code length is less than 11 bits. This parameter value is the same as that of Field 32 of the authorization.

Sending Institution’s Identifier Code

The code value is left justified, filled with spaces on the right if code length is less than 11 bits. This parameter value is the same as that of Field 32 of the authorization

Merchant Category

Acceptor’s Merchant Category Code (MCC)

Card acceptor terminal identifier code

The identifier of the card acceptor terminal

Card acceptor identification code

The identifier of the card acceptor assigned by the POS management authorities. This parameter value is the same as that of Field 42 of the authorization

Name and address of card acceptor

Name and address of the card acceptor

Filled with 0000.

Single or double message identifier

This parameter value defines if the message is single or double:

•0 - single message

•1 - double message

This value is set by the message sender according to the original transaction.

For offline purchase of IC card including debit/credit application and E-cash application, this field is filled with 0.

CUPS Retrieval Reference Number

A retrieval reference code filled by CUPS in Incoming STF files. It is filled in with 000000000 by Members in Outgoing.

Receiving institution’s code

A code of the financial institution receiving the message file. This parameter value is left-justified, filled with spaces on the right if code length is less than 11 bits. Its value is set in accordance with the original transaction.

Card Issuer’s Code

A code of the card issuer financial institution. This parameter value is left justified, filled with spaces on the right if code length is less than 11 bits. Its value is set in accordance with the original transaction.

Identifier of CUPS Notice

•0 - normal transaction record

Transaction initiating channel

Identifier of Transaction Features

The identifier of transaction characteristics. This field is filled with space for the following transaction types:

•F: full presentment of chargeback

•P: partial presentment of chargeback

Cross-border features tab

Parameter

Description

Point of service input mode code

A code of a card data input mode used at POS. This is the value of Field 22 in Part II Online Message

Authorization identifier (Floor limit)

An identifier of the authorization floor limit:

•0=online authorization

•1=authorization waived

•Others: Reserved for Use

For transactions in which the Transaction Amount is below a specified floor limit, online authorization is not compulsory. When submitting settlement files, acceptors set 1 for this parameter; if not applicable, the default value (0) is set

Type of payment service requested

This parameter determines a payment service use condition. It is a numeric two-bit code

Bit 1:

•0 = display of card. In this case, a card to which the payment service is allocated must be submitted when using this service

•1=non-display of card. In this case, a card to which the payment service is allocated must be submitted

•2=not given. The card submission is not specified

Bit 2: currently not applicable, filled with spaces

Settlement amount

The amount of the settlement. A numeric value of 12 bits

Settlement exchange rate

The currency rate used for settlement. A numeric value

Billing amount of cardholder

The billing amount on the cardholder’s side. A numeric eight-digit value. Bit 1: 0-7, referring to the position of the starting point from the right

Currency type of cardholder’s account

The code of the currency used for the cardholder’s account. A numeric value

Exchange rate of Cardholder’s billing

The currency rate used to calculate the cardholder’s billing amount. A numeric eight-digit value. Bit 1: 0-7, referring to the position of the starting point from the right.

Amount of services fee

Settlement amount of services fee after profit

RF Billing Currency

The code of the Reimbursement Fee billing currency.

For Reimbursement Fees that are collected by percentage of transaction volume (including capping and floor limits like POS), the currency of RF is the same as settlement currency. Therefore, the RF Billing Currency parameter value is set as ’Settlement currency’.

For the fixed fee per transaction, the currency of RF is RF Billing Currency

Exchange Rate from RF Billing Currency to Settlement Currency

The currency rate used to convert from RF Billing Currency to Settlement Currency.

For Reimbursement Fees that are collected by percentage of transaction volume (including capping and floor limits, like POS), the Currency conversion is not necessary. The Exchange Rate from RF Billing Currency to Settlement Currency is set as the fixed value ’30001000‘.

A fixed fee per transaction, for example, the currency of reimbursement fee for ATM is USD and the exchange rate for USD/HKD is 7.753, therefore, when the institution settles in HKD, the Exchange Rate from RF Billing Currency to Settlement Currency value is set as ’30007753’

Abbreviation of International Organization (corporation) and foreign institutions

An abbreviation indicating international organizations and institutions. The following values are possible:

VSA=Visa MST=MasterCard JCB=JCB AME=American Express DNC=Dinners Club, UPI=CUPS，JEC=Jetco，BCC=BOC Credit Card (International) Ltd.

For cross-border transactions with RMB cards and UnionPay cards, this field is filled with spaces

IC data tab

Parameter

Description

Applied Cryptogram

A cryptogram used for the card.

Represented with a 16-digit hexadecimal value (unmasked) (0–9, A–F); the value of TC is used

Point Of Service Input Mode Code

The card data input mode used at POS. This is the value of Field 22 in Part II Online Message.

Card Serial Number

A card serial number provided by the acquirer.

If the acquirer did not provide it, this field is filled automatically with '―000'

Read Capacity Of Terminal

A numeric code indicating the terminal capabilities for reading card data (according to terminal Technical Specifications)

IC Card Condition Code

The parameter value indicates whether magnetic stripe card transactions are permitted at the IC card terminal. The condition codes are set according to terminal Technical Specifications

Capacity Of Terminal

A code indicating the card data input, CVM and security capabilities of the terminal. Represented with a 6-digit hexadecimal value (unmasked) (0–9,A–F)

Result Of Terminal Verification

A code of a terminal verification result. Represented with a 10-digit hexadecimal value (unmasked) (0–9, A–F).

Unpredictable Number

A numeric value represented with an 8-digit hexadecimal value (unmasked) (0–9,A–F)

Serial Number Of Interface Device

The unique and permanent serial number assigned to the interface device by the manufacturer.

Issuing Bank Application Data

Information regarding the issuing bank application indicated with a 64-digit hexadecimal value (unmasked) (0–9, A–F). If the length of the effective value is less than 64, the subsequent content is blank

Application Transaction Counter

A value of the application transaction counter. Indicated with a 4-digit hexadecimal value (unmasked) (0–9, A–F).

Application Alternation Characteristic

This parameter value defined the characteristic of application alternation.

Indicated with a 4-digit hexadecimal value (unmasked) (0–9, A–F)

Transaction Date

The date of terminal authorization in the YYMMDD format.

Country code of the terminal

The code indicating the country of the terminal, represented according to ISO 3166

Script result of card issuer

Indicated with a 42-digit hexadecimal value (unmasked) (0–9, A–F). If the length of the effective value is less than 42, the subsequent content should be filled with blanks.

Transaction response code

The response code of the transaction:

•Y1 - successful offline transaction

•Y3 - cannot be online, successful offline transaction

Transaction category

The code of the transaction category. The value is the first two digits of field 3 (transaction handling code)

Authorized amount

The amount authorized for the transaction. This value is transmitted from the terminal. If such transmission fails, this field is filled with the default value; the 12-digit significant figure, of which the specific decimal digit number is subject to the authorized currency

Code of transaction currency

A three-digit significant figure referring to the transaction currency used for authorization

Cipher text information data

A two-digit hexadecimal value (unmasked) (0–9, A–F). If this field could not be filled, it is blank

Other amount

A 12-digit significant figure, of which the specific decimal digit number is subject to the authorized currency. If this field could not be filled, it is blank

Authentication method and result of the cardholder

A code of a cardholder authentication method and authentication result:

•1 - indicated with a six-digit hexadecimal value (unmasked) (0–9, A–F)

•2 - If this field could not be filled, it is blank

Terminal category

A terminal category code (two-digit significant figure). If this field could not be filled, it is blank

Dedicated document name

A name of a dedicated document. Indicated with a 32-digit hexadecimal value (unmasked) (0–9, A–F). If the length of the effective value is less than 34, the subsequent content is blank

Application version No

The number of the application version. Indicated with a two- digit hexadecimal value (unmasked) (0–9, A–F)

Transaction serial counter

A value of a transaction serial counter. If the length of the effective value is less than 8, the subsequent content is blank
