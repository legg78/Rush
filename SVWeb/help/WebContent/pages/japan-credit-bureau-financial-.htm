Japan Credit Bureau financial messages

The Operations page displays financial messages generated when processing Japan Credit Bureau authorizations.

1.Select Payment System > Japan Credit Bureau > Financial messages.

 

2. Specify the search parameters in the search pane:

Parameter

Description

Operation ID

The transaction identifier.

Card number

The masked card PAN.

Date From

Transaction execution period.

Date To

Calculation status

The transaction calculation status:

•Waiting

•Calculated

•Failed

•Transferred

3. Click Search.

The search result pane displays a list of messages according to the specified conditions:

Parameter

Description

ID

The transaction identifier.

Session ID

An identifier of a file load session.

Card number

The masked card PAN.

Operation amount

The transaction amount in the transaction currency.

Settlement amount

The transaction settlement amount.

Date

The transaction execution date.

Issuer Inst ID

Issuing institution identifier.

Acquirer Inst ID

Acquiring institution identifier.

Calculation status

The transaction calculation status:

•Waiting

•Calculated

•Failed

•Transferred

Fees count

Total amount of transaction fees

Additional information about a selected message is available in the Details and JCB tabs under the list of messages.

Details tab

Parameter

Description

Acquirer Inst BIN

BIN assigned to the acquiring institution.

Acquirer Network ID

Identifier of the acquirer network that participated in the transaction.

Issuer Network ID

Identifier of the issuer network that participated in the transaction.

Host Date

The date of the transaction as determined by the host.

Issuer Card Country

Country in which the transaction card was issued.

Is Reversal

Indicates whether or not the transaction is a reversal.

MCC

Category code of the merchant where the transaction was performed.

Merchant Country

Country in which the merchant is located.

Merchant Region

The subnational region within which the merchant is located.

Merchant City

The city in which the merchant is located.

Merchant Street

The street in the city in which the merchant is located.

Merchant Postcode

The postal code of the city in which the merchant is located.

Merchant Name

The merchant’s name.

Merchant Number

The number identifying the merchant.

Message Type

The type of the message according to the MSGT dictionary.

Network Refnum

Retrieval reference number.

Operation Request Amount

The requested transaction amount.

Operation Type

Type of the transaction.

Status

The message status according to the CLMS dictionary. The message status is determined by the MESSAGE_STATUS parameter in the Create JCB financial message procedure configuration (see Viewing rule set procedures).

Settlement Type

The settlement type specifying the type of transaction

Terminal Number

The number identifying the terminal.

Terminal Type

The device type for which the template is to be used.

JCB tab

The JCB tab contains a list of selected message fields according to the JCB message specification and their values.
