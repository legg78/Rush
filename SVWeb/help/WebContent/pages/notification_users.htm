Introduction to notifications

User notifications are configured similarly to customer notifications. The only difference is in the application of notification schemes, which define a set of events for which notifications will be sent (for more information, see Notifications).

A notification scheme can be created for any role (scheme type – Customer notification). This scheme contains a list of objects events for which a notification must be sent.

A list of the objects for which the notification can be sent is limited by the role privileges to view these objects.

Note: Notification objects are displayed on the Custom events.

A user can change the notification parameters for themselves: disable notifications, edit objects filtering, etc. This rule replaces the rules configured for the role:

•If both parent role and subrole have notification schemes (the same or different), and the same event notification is set in these schemes, then the rule set for the parent role will operate for the user (who has those two roles).

•If two different and not hierarchically related roles are assigned to the user, and these roles have different notifications schemes for the same event, the user will receive two notifications per the event.

•If two different and not hierarchically related roles are assigned to the user, and these roles have the same notification scheme for the same event, the user will receive a single notification.

Note: Only one notification scheme can exist for each processing member.
