Specifying process files in container

Specify the Load priority, Signature type, and Encryption plugin parameters for secure file exchange.

1.Click Administration > Processes > Containers.

2.Specify search parameters on the search pane and click Search.

3.Select the required container.

4.On the details pane, click the Files tab.

5.Select the record and click Set value.

6.Specify the file parameters.

Parameter

Description

Location

Path to the input/output file which is set in the Administration > Processes >Directories

Converter class

Java class for conversion

Name algorithm

Algorithm used to generated the file name

Character set

Character encoding (for example, UTF-8).

Line separator

Specifies the character used to separate lines in the file:

•OS-defined - defined by the operating system

•Unix - line separators used in Unix systems

•Windows - line separators used in Windows

Load priority

File order for loading; signature file must be first, before the data file

Signature type

Specifies signature attachment method:

•no signature

•signature in a separate file

•signature together with the file

Encryption plugin

Java-class for input/output signature processing. If there is no signature, the parameter is disabled.

TAR

A check box that indicates that the input file is a UNIX tar archive to be uncompressed, or that the output files must be compressed into a tar archive.

ZIP

A check box that indicates that the input file is a ZIP archive to be uncompressed, or that the output files must be compressed into a ZIP archive.

Unique file name

File name must be unique.

Upload empty file

Empty file generation is available.

Ignore file errors

Instructs the system to ignore file-processing errors

File is required

Specifies whether the file is mandatory or optional

Report

The report to be generated for the event to which the process is subscribed.

Report template

Template to be applied to the specified report

Password protect

Parameter enabling password protection for the report file. This option is available if the report is specified in the Report field on this page..

If this check box is selected, running the container results in the following actions:

•the report file is secured by a randomly-generated password

•the system generates a notification message containing this password and sends it to the customer's mobile number

XSLT source

This button displays the XSLT source files for the XML files in the configured system viewer or browser.

Applying XSLT templates enables you to transform one XML files into other XML file types, or into other formats such as HTML, text, or XLS Formatting Objects. The converted file may subsequently be converted into other formats.

Parallel degree

Number of threads to process the file

Queue name

Name of the MQ  to which the file is placed.

Time wait

Time, specified in seconds, that the process waits for data or for a response.

For output files you may specify a report that is generated for the XML file.

7.Click Save.
