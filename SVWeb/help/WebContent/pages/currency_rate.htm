Exchange rates

The SmartVista system uses exchange rates in order to convert one currency to another during calculations. For example, a fee may be specified in dollars, while the transaction amount and account are denominated in euros. In this case it is necessary to convert the dollar fee to a euro amount before debiting the relevant account.

Note: Each institution has its own set of exchange rates.

An exchange rate has an associated rate type, which defines the applicability of the rate (for example, whether it is an internally specified rate, an interchange rate, a commercial rate, or a rate set by the Central Bank). The system maintains rate type names in a single dictionary (RTTP) common to all institutions.

Exchange rates and their application rules (for example, the rounding accuracy to be applied, or whether the rate can be inverted for reverse exchanges) are defined for each institution separately. A rate should be set up for every rate type registered for each institution (source and destination currencies defined, their ratio set, scales an dates given, etc.).

A new rate is always registered with Valid status. This status allows the system to use the rate for conversion. If a rate is changed to Invalid status it can no longer be used by the system.

Rate conversion requires specifying the rate type (for example, in processing rules, service terms). The necessary rate is applied depending on the institution for which a transaction is processed.

If a transaction currency differs from a limit/fee currency or from an account currency, then the applicable exchange rate is used.

A rate pair (rate template) could be registered along with a rate type. According to the template parameters a new dependent rate is created automatically after that if you register a rate that has a rate pair for the same currency pair. This rate is calculated based on a formula set for the dependency of these rate types (see the topic describing rate pair setup).
