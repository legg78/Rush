Manually launching a container

Containers can be run manually by users provided they are assigned the necessary role configured in the container.

1.Click Administration > Processes > Containers.

2.Specify search parameters on the search pane and click Search.

3.Select the required container on the search result pane.

4.Click Launch.

5.In the Launch window, specify the mandatory and additional process parameters.

6.Click Launch.

In the container launch page, you can configure Oracle tracing for the selected container or its process threads. This enables the container process performance to be monitored and analyzed, which may be required if the process is taking an excessively long time to execute. With Oracle tracing enabled, a user can create a process trace log and submit it to the system administrator for further analysis.

When the process is finished, the Oracle tracing is disabled automatically. A user can switch off Oracle tracing on the Statistics tab of the Monitoring > Process log page using the Oracle tracing button (see the Viewing process logs topic). It is important that Oracle tracing is only activated for a limited time in order to avoid large trace logs adversely affecting system performance. For this purpose, a DBMS administrator can also specify the maximum size of the Oracle trace file.

To enable Oracle tracing for the running container:

1.Click Oracle tracing and specify the tracing parameters:

 

Parameter

Description

Session ID

Number of the session in which the container is launched.

Trace level

The level of the tracing information available in the trace file:

•0 - No trace - tracing is disabled

•2 - Simple tracing - standard trace information without any additional data (the default value)

•4- Tracing with bind variables- values of variables used in SQL operators are added. This information is lost after processing the trace file

•8 - Tracing with waits - information on wait events is added

•12 - Tracing with waits and bind variables - all information including levels 2, 4 and 8

Trace limit

Period in minutes for the Oracle tracing (the default value is 10 minutes, which avoid using excessive disk space).

Thread number

Number of process threads within the container for which Oracle tracing will be applied.

2.Click Save.
