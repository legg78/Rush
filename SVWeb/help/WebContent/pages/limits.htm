Limits

Limits are used to calculate actions, transactions, events, etc. The various actions are dependant on the counter value, they can be: a fee charging, an object blocking, etc. Service terms of this type can also be used to restrict/ban something (for example of the term: the limit of cash withdrawal).

Possible limit values are:

•0 — prohibitive limit (for example, a value of 0, which was set for the Card cash withdrawal cycle limit service term, prohibits the withdrawal of card funds from any ATM)

•Unlimited — unbounded limit. It is used to indicate that the limit on the counter is not installed

•Any positive number

Limits can be cyclic. This means that after the cycle is completed the limit counter is reset and accumulation begins again (in a new cycle). For a non-cyclic limit, the counter limit counter is increased steadily and never resets. If the cycle is not specified, it is assumed that the limit is non-cyclic.

The limit counter is changed when an event relevant to the limit type occurs. The limit counter is also changed during operations. In this case, a rule set that defines a transaction processing method includes the procedure for the limit accumulation.

SmartVista saves information about all limits by all operations (on transaction or authorization). This information is necessary for the correct processing of the full or partial transaction or authorization cancellation. On transaction cancellation processing, the limit counter is reduced by the amount saved in history of the transaction, or by the amount proportional to the sum of cancellation. When the limit cycle ends the counter resets and the counter changing history is cleared. If there is no record in the limits history for a transaction, the cancellation reduces the current limit counter. Since this means that the limit counter has been reset before the information about the transaction cancellation was received.

Limits with unidentified extreme values and which are defined as the proportion (percentage) of another limit, are referred to as dependent limits. The extreme value of such limits is not stored but is calculated dynamically during each scan (counter increasing).
