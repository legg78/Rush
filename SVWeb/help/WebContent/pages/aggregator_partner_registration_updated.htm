Aggregator/Partner registration procedure

This section describes the steps that are required to register an aggregator/partner. To register an aggregator/partner, a new institution must first be created and then added to the banks local network. The institution must be set as the host owner and its authorization interface must be defined.

 

The bank institution must then be added as the host customer and the standard parameters for settlements must be configured. The customer (bank institution) will represent the Aggregator/partner and has to be registered via the issuing application.

Creating the institution

1.Create the institution (institution type: Aggregator, Partner or Aggregator/Partner) to be used as the aggregator/partner.

Do not enter anything into the Default network field.

2.Add the newly registered institution into the bank local network (on the Members tab of the Network page).

3.If you are registering an acquirer type participant (PRTYACQ in dictionary code) select 'Acquirer' from the Participant type list. If you are registering an aggregator type participant (PRTYPAGR in dictionary code) select 'Payment aggregator' from the Participant type list

4.Go to the Hosts page and add a new host. Select the new institution as the host owner.

The host status corresponds to the partner status.

1.Set the connection parameters:

a.For the host (aggregator), which will be used for payments via the institution, specify the Authorization interface: Aggregator Gate Protocol

b.For the host (partner), which will be used to receive the payments via the institution, specify the Authorization interface: SVIP-IFX Host Standard (which is defined by the partner)

2.Set the version of the standard.

3.Add the bank institution as the host customer and configure the standard parameters for settlements.

4.Add the device (on the Connectivity tab of the Hosts page). Select the device from the available ones or create a new device by selecting HTTP Communication from the Communication plugin list.

5.Configure the connectivity parameters of the standard (the Configure button at the Connectivity tab).
 

6.Register the customer via the issuing application (use ОENTTCOMP – Organization customer type). This customer represents the aggregator/partner (Business-process: the issuing application by default.
