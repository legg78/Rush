Setting a service term value

You can set a service term value at the level that was specified as the definition level of the term. The service term value can be set during an application registration provided the term definition level is either an object or a product.

To set a service term value:

1.Access the page that corresponds to the service term definition level.

The table below shows the accordance of service term definition level with the page, on which the setting is performed.

Definition level

Page to set a service term value

Service

Configuration > Servicing > Services.

Product and object

For a Product, click Issuing/Acquiring > Configuration > Products.

Object

Service terms at an object definition level are set at different pages depending on the object type required.

For Account type objects, click Issuing/Acquiring > Accounts.

For Card type objects, click Issuing > Cards.

For Terminal type objects, click Acquiring > Terminals.

For Merchant type objects, click Acquiring > Merchants.

For Customer type objects, Click Customers > Customers.

For Contract type objects, Click Customers > Contracts.

 

2.Click Search on the search pane.

3.Select the service/product/object (for example, Card, Account, Terminal) to set the service term value.

4.Click the Service terms details pane tab.

A warning icon means that the service term cannot be set on this page (level):
 

5.Select the service term and click Set.

Note: The window view differs depending on the service term type (for example, cycle, fee, limit).

Note: If the scale was set for the service term, the Modifier list is displayed below the period of validity lists. You can select the previously created condition on which the service term value is applied.

6.Specify the parameter value.

7.Click Save.
