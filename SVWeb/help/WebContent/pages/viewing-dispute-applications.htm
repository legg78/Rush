Viewing dispute applications

To view dispute applications registered in the system:

1.Click Operational rules > Disputes > Applications.

2.Specify the search parameters on the search page.

Parameter

Description

ID

Application internal number assigned by the system

Card number

Number of the card used for the disputed transaction. In the main table, the masked number is displayed. Clicking the card mask link opens the cardholder’s contacts.

Branch ID

Unique identifier of a branch to which the claim was submitted. This identifier is assigned according to the bank internal classification. A user from a particular branch can view the applications registered by this particular branch users only.

Date from

Application creation period

Date to

Status

Application processing status:

•New

•Ready for review

•Accepted

•Rejected

To display a full list, leave the fields empty.

3.Click Search.

The search result pane displays a list of registered notification schemes with information about their institution, types, and text description.

Additional information about a selected record is displayed on the Details and Documents tabs.

The Details tab contains the disputed transaction information:

Parameter

Description

ID

Application internal number assigned by the system

Card number

Number of the card used for the disputed transaction

Created

Date when the application was created

Status

Application processing status:

•New

•Ready for review

•Accepted

•Rejected

Transaction date

Disputed transaction date

Currency

Disputed transaction currency

Amount

Disputed transaction amount

Reject reason

Application rejection reason according to the Application reject statuses dictionary:

•The claim was submitted before

•The disputed transaction was not found in the system

•The disputed amount was already credited to cardholder's account

•The required information was not specified (specify)

•The required supporting documents were not enclosed (specify)

•Other (specify)

This field is displayed only for rejected applications.

Reject comment

Reject comment text that specifies the reason the dispute application was rejected. This field displays for rejected applications only.

The Documents tab contains data on supplementary documents submitted with the application:

Parameter

Description

ID

Document identifier number

Document type

Supplementary document type:

•Cardholder dispute form (a mandatory document that must be attached to the application)

•Copy of sales slip

•Copy of credit slip/voucher

•Copy of return receipt

•Copy of alternate payment

•Copy of merchant’s delivery terms and conditions

•Correspondence with merchant

•Other attachment regarding a dispute

File

File attached to the application

Document date

Date upon which the document was created
