Setting the system audit level

1.Click Administration > Settings.

2.In the Tracing section, select the Trace level parameter and click Edit.

3.In the Parameter setup window, select the trace level and click Save.

4.Similarly, set the required values for the other tracing parameters:

Parameter

Description

Trace into dbms_output

Alternative method that is usually used for code debugging. The dbms_output file contains a formatted string of the same parameters that are stored in the table.

Trace into log table

Principal method for storing trace messages. An autonomous transaction procedure is responsible for saving messages in the table.

Trace into session

The package dbms_application_info is used for this type of tracing. This package enables the last message parameters to be recorded as the current session status.

Debug info writing mode

The mode of recording debugging messages into the log:

•LGMDIMDT - Immediate saving

•LGMDNRRR - Or error

•LGMDSSPD - Suspended recording

Trace for first debug count

Defines the number of code debugging messages for the trace log. The default value is 0 (inactive). A positive integer specifies the number of messages that must be added to the trace log. These messages must have a tracing level lower than the current system tracing level. For example, if the Trace level is set to Error and the Trace for first debug count is set to 500, the trace log will include not only errors and fatal errors, but also the first 500 messages of the Warning, Info, and Debug levels.

Trace for last debug count before error

Defines the count of code debugging messages before an error message is generated. The default value is 0 (inactive). A positive integer defines the number of the last debug messages with a debug level lower than the current system one that will be recorded with an error or fatal error message. The value of the Debug info writing mode parameter must be On error.

Debug messages follow the error messages (if any error messages are available).

For example, assume that the Trace level is set to Error and the Trace for last debug count before error is set to 100. If an error occurs, the trace log will include 100 debug and info messages that will precede error messages.

The new value of the audit level takes effect immediately for all institution objects. It will also be applied to those institution users who do not have an audit level explicitly set.
