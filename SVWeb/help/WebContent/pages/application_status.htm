Application flows

An application flow defines a set of its statuses (stages) and possible transitions between these statuses. An application must pass through all the statuses until final processing. The final state of any application is 'success'.

Some statuses are required for the life cycle of the application and they should be present in all processes of the application’s registration. Other states depend on the specific business processes of the organization.

The statuses which are mandatory for applications are listed in the table below.

Status

Description

Just created

The default status of any newly created application. During the creation of applications this status may be left the same or changed to another specified in the application flow.

Awaiting processing

An application can be switched to this status from the previous one. Processing of applications is only possible in this status. If this status is not included in the registration flow, the application cannot be processed.

Processed successfully

This status is assigned to the application after it has been successfully processed. This is the final status of the application. Applications in this status can only be viewed.

Processing failed

This status is assigned to the application when an error occurs during the application processing. In this case, the screen displays an error message. The application in this status should be corrected and processed again. After the correction, set the application to the initial status.

When an application passes through the status Processed successfully, changes occur in the system, for example,  a new customer is created, a new account is opened, a new instance of the card is issued, etc.

Note: Depending on the business processes of the organization, there may be other statuses between the Just created and the Awaiting processing statuses.

It is recommended that the transition status be planned as it enables the operator to continue working with the application in the case of an error. In other words, the operator can enter a part of the data for the first time, then save the data, while leaving the application in the same status. Data entry can be completed later by switching the application to the next status.

The transitions between the Awaiting processing — Processed successfully statuses and the Awaiting processing — Processing failed statuses run automatically when you click Process or process the application via a container. It is not necessary to configure these transitions for a flow. The transitions between the four mandatory statuses (Just created, Awaiting processing, Processed successfully, Processing failed) are adjusted according to the business processes of the organization.
