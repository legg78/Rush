Common criteria for report data selection

Data for quarterly reports is selected according to the requirements listed below.

Note: The types of cards, transactions and settlements and their codes are provided in the relevant system dictionaries, which can be configured according to the requirements of the financial institution.

Reported data

Criteria

Card type

Card types are determined by the card product characteristics specified in the following parameters:

Feature (see the Card types topic)

MasterCard quarterly report card types and Visa quarterly report card types (see Reference information > Array types and arrays and Lists of values)

Card types are configured according to the international standards for Visa and MasterCard respectively (Visa International Operating Regulations and MasterCard Rules).

Settlement type

Settlement types are configured for issuing and acquiring settlement by the Iss Settlement types and Acquirer settlement types arrays according to the international standards for Visa and MasterCard respectively (Visa International Operating Regulations and MasterCard Rules). For more information, see Reference information > Array types and arrays.

Transaction type

Transaction types are configured for issuing and acquiring settlement types according to the international standards (Visa International Operating Regulations and MasterCard Rules). The types of transactions to be included into quarterly reports are defined by the QPR Operations and QPR Operation status arrays. Array conversion rules establish correspondences between the specified array elements and types of transactions, available in the system. For more information, see Reference information > Array types and arrays.

Number of cards

Valid cards within their start and expiry dates that are available to be used at any time to perform a transaction. Card validity is also defined according to its activity status.

Active cards:

Valid cards that have been used for at least one transaction during the report quarter.

Inactive cards:

Expired cards

Blocked cards

Lost and stolen cards that have been blocked

Year, quarter, institution

These are common parameters that are used in most reports. The required year and quarter are numeric values, which must be entered manually. The institution is selected from the list of institutions configured for a particular installation.

Examples of parameter values:

•Year — 2015

•Quarter — 1

•Institution ID — 1001 - Processing center

Note. If Multi-institution  = 0 - No (Administration > Settings > Settings) and the parent institution is selected, data on all its affiliates will be aggregated in one report.
