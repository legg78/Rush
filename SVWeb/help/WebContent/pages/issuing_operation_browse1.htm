Viewing issuer transactions

1.Click Issuing > Operations.

The search pane has the following tabs:

Tab

Description

Operations

Search all transactions in the database.

Document

Search transactions with documents (for example, primary and payment documents) that were assigned to transactions or payment orders.

Participant

Search all transactions using participant parameters.

Payment order

Search transactions with payment orders.

Tags

Search for transactions by predefined transaction tags.

2.Click the required tab and specify the search parameters. Example for the Operations tab:

•In the Host date from and Host date till fields, specify the date of transaction as determined by the host.

The Merchant number and Terminal number fields are not used in issuing.

•In the Operation ID field, specify the desired transaction identifier.

•From the Status list, select the current status of the transaction.

Leave the fields empty to list all registered transactions.

3.Click Search.

The search result pane displays a list of the transactions. The following icons are used:

• — canceled transaction

• — reversed transaction

The details pane tabs display the selected transaction data.

Tab

Description

Details

General information about the transactions.

Accounting

Debit/credit account, balance types, debit/credit amounts. The data can be viewed in simplified mode if the Simplified view check box is selected.

Payment order

Information about the payment order to which the transaction corresponds.

Trace

The transaction processing steps displayed in accordance with the specified trace level.

Tags

Selected transaction tags. The list of available tags can be found on the Tags page (see Operational rule > Authorization > Tags).

Authorization

Information about the authorization of the transaction.

Participants

Information about issuing and acquiring objects related to the transaction: institution, card type, account number, amount, account currency, etc.

Messages

System messages that are generated during transaction processing.

Disputes

Information about any disputes that may have occurred during transaction processing.

A dispute is initiated based on the original transaction to verify the validity of the original transaction.

Additional amounts

Transaction amount.

Notes

Notes related to the selected transaction (such as user comments, delivery notes, or notes relating to account opening and closing).
