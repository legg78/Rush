Configuring custom events

A user notification for users currently operating the system can be configured for any event. A list of notification objects that can be set is limited by the role privileges assigned to that user. For example, if a card is issued (event), the user receives a notification, typically by email, in the specified time interval. The Custom events tab of the User settings page displays all event types for which a user can obtain notifications.

If permitted, a user can change notification parameters (for example, delivery time or notification method).

1.Open the User settings page.

2.Select the Custom events tab.

3.Select the event type from the list to change the notification rule. If the rule is not editable, the Set button is inactive.

4.Click Set.

5.In the Add new custom event window, edit the event parameters (not all parameters are editable).

Parameter

Description

Channel

Delivery method for notifications.

Delivery address

Address of the contact.

Delivery time

Time during which the notification may be sent. If the event occurs outside this period, the notification is delivered during the specified interval the next day.

Modifier

Determines whether additional conditions are required, e.g. Notification delivery. These can be configured by editing the modifier conditions.

Status

Notification status:

•Active

•Not active

6.Click Save.

Note: Configuring custom events is dependent on the configuration of notification schemes.
