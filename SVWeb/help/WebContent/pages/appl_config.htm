Configuring applications

All structural changes to business objects in the SmartVista system are processed through either an issuing or acquiring Application entity.

The application structure is hierarchical and can be extended. The hierarchy is built with information blocks that are grouped by the system entities and logical groups. Several similar blocks may be contained in one block (these blocks are characterized by a sequence number). At the same time, similar parameters can be included in a single block (these parameters are characterized by the sequence number).

Blocks and parameters can be mandatory or optional. The mandatory feature for parameters and blocks depends on  the action that the system performs on block processing.

Each block contains a Command parameter that defines the system behavior depending on the presence or absence of the object in the system:

•Create or except / proceed / update

•Except or proceed / remove / update

•Ignore

•Proceed or remove

The application is created with the Just created status. This is the initial status of any application. During the application processing, its status is changed according to the predefined rules of the application flow. When the application passes all the business stages and can be processed, its status is then changed Awaiting processing.

Working with the application may comprise a more complex sequence of actions, rather than a one-time entry of all the data by one operator. For example, different data can be set by different employees in a particular sequence in accordance with their job responsibilities and the company’s business processes. In this regard, the term application registration is used.

During application processing, SmartVista generates the required parameter values ​​(for example, identifiers, numbers of cards, accounts, etc.) which are stored in the structure of the application.

If an error occurs during the processing the corresponding error message is logged. The message contains information about the application identifier (as the entity identifier) and the block identifier (as the identifier of the event that caused the processing the error).
