MasterCard Financial Messages

The Financial Messages page displays financial messages generated when processing of MasterCard authorizations.

1.Select Payment System > MasterCard > Financial messages.

2.Specify the search parameters on the search pane.

Parameter

Description

Session ID

An identifier of a file load session (filled manually).

File name

A name of a message file (filled manually).

Rejected

an authorization rejection sign. Available values (selected from the list):

•0 - No — messages for accepted authorizations are displayed

•1 -Yes — messages for rejected authorizations are displayed

Date from and Date to

The initial and final dates of the period for which messages are displayed (selected in the calendar).

To display all available messages, leave the filter fields empty.

3. Click Search.

The search result pane will display a list of messages according to the specified conditions. The following data is available for each message:

Parameter

Description

Status

A message status (according to the CLMS dictionary). The message status is determined by the MESSAGE_STATUS parameter in the Create MC financial message procedure configuration (see Viewing rule set procedures).

D

Message direction (incoming or outgoing).

R

A reversal sign.

Card

Card number (PAN).

MTI

Message type identifier.

DE038

Approval code.

DE004

Amount requested by the cardholder.

DE005

Reconciliation amount.

DE043_1

Card acceptor name.

Session ID

Identifier of the session in which the file was loaded.

File name

Name of the message file (a link to the file of the file containing the selected message in the Files window).

Additional information about a selected message is available in the following tabs:

Details tab

Parameter

Description

General data

General data on the selected message.

DE0

The values of DE0 (Data Element) type fields in a message. For more details, please refer to IPM Clearing formats.

EMV

The values of EMV-compliant fields. For more details, please refer to IPM Clearing formats.

P0

The values of Private Data Subelement Definitions (PDS). These values are identified with the prefix P followed by the four-digit tag. For more details, please refer to IPM Clearing formats.

Addendum tab

The Addendum tab contains additional information about the selected message (such as a message type (outgoing or incoming), message type identifier and values of different message fields. Field codes are given according to the MasterCard message file specification).

Reject tab

The Reject tab contains rejection details (for rejected authorizations). Field codes are given according to the MasterCard message file specification).

Reject codes tab

Parameter

Description

DE number

The data element code.

Severity code

The severity value (for messages received from ATM).

Message code

A rejection message code.

Subfield ID

The subfield number.

Buttons

Button

Description

Modifying the selected record in the list of messages.
