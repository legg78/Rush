Service provider registration

During a payee registration a Service Provider entity is registered and associated with the corresponding customer. An issuing application is used for this procedure.

For this application the following parameters are applied:

•Customer type: ОENTTCOMP – Company

•Flow: Default Issuing application

•Customer extended type: ENTTSRVP – Service provider

On the customer level, a contract of the Service provider type is opened. The contract defines payments to the service provider (for example, Minimum and Maximum payment amount). These parameters are represented by service terms.

Use the Service terms tab (Customer page) to set up service terms for the customer.

Provided services for which payments are carried out, are registered as services (see the Services page).

 

To display the list of registered service providers:

1.Click Payment orders > Service providers.

2.Specify search parameters and click Search.

The search result pane displays the list of registered service providers, and details pane displays general and additional information about the selected one: provider’s name, purpose and the region code (Details tab), assigned host (Payment hosts tab), services of that will take payments through this host (Services tab).

To register a service provider:

1.Click Payment orders > Service providers.

2.Click Add. This brings up the New provider window.

3.Specify the provider parameters.

•From the Language list select the language of the input text data

•In the Name field, type the service provider name as it will be displayed in user interface

•In the Region code field, set the host region code. This parameters is used on record sorting only

•In the Short name field, type the short name (abbreviation) of the service provider name

•In the Description field, type any additional information about the service provider or its host

4.Click Save.

5.Set up the provider’s host(s) on the details pane (Payment hosts tab).

6.Specify the provider’s services (Services tab).
