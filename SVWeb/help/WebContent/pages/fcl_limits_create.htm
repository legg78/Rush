Setting the limit value

The user sets a limit value on the level that was specified as the definition level of the corresponding service term: service, product and object, or object. A limit value must be specified for processing to proceed correctly.

1.Access the page that corresponds to the service term definition level.

Definition level

Page to set a service term value

Service

Configuration > Servicing > Services.

Product and object

•Product: Issuing/Acquiring > Configuration > Products.

•Account type object: Issuing/Acquiring > Accounts.

•Card type object: Issuing > Cards.

•Terminal type object: Acquiring > Terminals.

•Merchant type object: Acquiring > Merchants.

•Customer type object: Customers > Customers.

•Contract type object: Customers > Contracts.

Object

•Account type object: Issuing/Acquiring > Accounts.

•Card type object: Issuing > Cards.

•Terminal type object: Acquiring > Terminals.

•Merchant type object: Acquiring > Merchants.

•Customer type object: Customers > Customers.

•Contract type object: Customers > Contracts.

2.Click Search on the search pane.

3.Select the service, product, or object (for example, Card, Account, Terminal) to set the service term value.

4.Click the Service terms details pane tab.

A warning icon means that the service term cannot be set at this page (level):
 

5.Select the service term of the limit type and click Set.

6.Specify the limit parameters value on the corresponding tab.

oSelect the radio button that defines how the value is specified:

Value

Description

Create new value

Set all parameters of a new value.

Clone value

Duplicate the previously set value, all parameters of the limit value can be edited except the Count limit and Sum limit parameters.

Apply value

Duplicate the previously set value, only the period of validity can be edited (the Start date and End date fields).

oFrom the Limit list, select the limit to duplicate its value. The box is available if the Clone value or Apply value radio button was selected. After the limit is selected, the value parameters are set in the following fields

oIn the Count limit field, specify the maximum number of operation and transactions. Possible values are the following:

Value

Description

Zero value (0)

Prohibitive limit (for example, a value of 0 specified for the Card cash withdrawal cycle limit service term prohibits the withdrawal card funds from any ATM).

Unlimited

Unbounded limit. It is used to indicate that the limit on the counter is not specified. To set the value, click . It is displayed on the right of the box.

Any positive number

Any number greater than 0.

oFrom the Currency list, select the limit value. If the currency is not set, then the account/BIN currency is set for an account/card accordingly. For all other entities, the limit currency must be specified explicitly.

IMPORTANT: For a fee with a limit: the fee and limit currencies must be the same.

Note: If the sum and count limits were specified then the two counters will be processed. And if one of them is exceeded, then the operation is canceled.

oIn the Sum limit parameters specify the limitation on the operation sum. Possible values are the following:

Value

Description

Zero value (0)

Prohibitive limit (for example, a value of 0 specified for the Card cash withdrawal cycle limit service term prohibits the withdrawal card funds from any ATM).

Unlimited

Unbounded limit. It is used to indicate that the limit on the counter is not specified. To set the value, click . It is displayed to the right of the box.

Any positive number

Any number greater than 0.

oFrom the Limit base list select the core limit that will influence on the current one (this parameter is used for a dependent limit).

oIn the Limit rate field specify a percent rate of the core limit (this parameter is used for a dependent limit).

oIn the Start date and End date fields, specify the limit validity. If the end date is not set then the limit has an indefinite validity period.

Note: If the scale was set for the service term then the Modifier list is displayed below the period of validity combo boxes. The user can select the previously created condition on which the service term value is applied.

For the description of the Fee and Cycle tabs, see in the corresponding chapters on fees and cycles.

7.Click Save.
