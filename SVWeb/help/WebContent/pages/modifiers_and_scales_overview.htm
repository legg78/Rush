Scales and modifiers

A modifier is a generalized method for parameterization of objects, settings, and events in the SmartVista system. In other words, a modifier is a set of conditions (rules) by which the value of a parameter is set (For example, the fee rate value may vary depending on the type of transaction).

The Oracle PL/SQL language is used to construction these terms and so a basic knowledge of PL/SQL is recommended in order to work with modifiers.

Each modifier belongs to a certain scale. This scale, in turn, is characterized by a scale type. A scale type defines its purpose and limits the list of scales in the which can be assigned to any object.

Note: The system comes with predefined types of scales that cannot be changed (non-editable dictionary).

 

Modifier terms can use the parameters specified for the scale only (for example, the message type or the type of calculation). Terms of modifiers within the same scale should be set so as to cover the entire range of possible situations.

To configure a modifier do the following:

1.Create a scale

2.Specify the scale parameters

3.Create a modifier with the parameters defined on the previous step.
