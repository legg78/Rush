Creating a new terminal template

A terminal record get its values from the terminal template that is used to create it. However, other values may be specified during application processing to override the values from the template.

1.Click Administration > Communication > Terminal templates.

2.Click Add on the search result pane.

3.Specify the template parameters (all mandatory):

Parameter

Description

Language

Language of the input text data.

Name

Terminal template name.

Description

Any additional description of the template.

Institution

Institution to which the the template belongs.

Terminal type

Device type to which the template applies.

Standard

Terminal communication standard.

Status

Default terminal status for the template.

GMT offset

Time zone applicable to the terminal location.

Use MAC

Specifies whether the message authentication code (MAC)functionality is to be used in transactions.

MCC redefinition group

Group that defines the rule of the terminal MCC (merchant actegory code) redefinition. The MCC redefinition functionality enables the MCC to be specified based on the transaction type for a specific terminal. MCC redefinition enables the system to detect an MCC for a transaction on a terminal and transfer it to the payment network. The MCC is necessary in order to process a transaction correctly.

Card data present

Specifies whether a physical card is required to initiate transactions from the terminal.

Card data input capability

Method of obtaining a card data:

F2210000 — Unknown; data not available.The default value

•F2210001 — Manual; no terminal. This value means that card data can be entered only manually

•F2210002 — Magnetic stripe reader capability

•F2210003 — Barcode reader

•F2210004 — Optical character reader (OCR) capability

•F2210005 — Integrated circuit card (ICC)capability

•F2210006 — Key entry-only capability

•F221000A — PAN auto-entry via contactless magnetic stripe

•F221000B — Magnetic stripe reader and key entry capability

•F221000C — Magnetic stripe reader, ICC and key entry capability

•F221000D — Magnetic stripe reader and ICC capability

•F221000E — ICC and key entry capability

•F221000M — PAN auto-entry via contactless M/Chip

•F221000V — Other capability. This value means that there is a data input method other than those available in the list.

Card data input mode

Device from which the data should be read.

Card capture capability

Specifies whether the terminal can capture a card.

Card data output capability

The method of recording data to cards:

•ICC — the terminal can update data on the chip.

•Other — the terminal can update the card data using a means other than the ones listed.

•No — data cannot be written.

Terminal operating environment

Terminal environment:

•F2240000 — No terminal used

•F2240001 — On card acceptor premises; attended terminal

•F2240002 — On card acceptor premises; unattended terminal

•F2240003 — Off card acceptor premises; attended

•F2240004 — Off card acceptor premises; unattended

•F2240005 — On cardholder premises; unattended

•F2240006 — Off cardholder premises; unattended

•F2240007 — Private use (future use)

•F2240009 — Unknown; data unavailable

•F224000A — Attended cardholder terminal on card acceptor premises

•F224000B — Unattended cardholder terminal on card acceptor premises

Terminal data output capability

Specifies whether information about the terminal is displayed on transactions. If it is, then you should specify where to display the information.

Available operations

Transactions that can be performed at the terminal.

Cash dispenser present

Specifies whether the terminal has a dispenser.

Use card possibility

Specifies whether the transactions from the terminal can be performed with cards.

Terminal profile

Terminal profile value. A terminal profile is a set of values of terminal parameters predefined in the Front End system.

Cardholder data present

Specifies whether the cardholder is required to be present to perform a transaction.

Cardholder authorization capability

Specifies whether the terminal supports cardholder authorization.

Cardholder authorization method

Method of cardholder authorization.

Cardholder authorization entity

Method used for cardholder's authentication on the terminal:

•F2290000 — Not authenticated

•F2290001 — ICC - offline PIN

•F2290002 — Card acceptance device (CAD)

•F2290003 — Authorizing agent - online PIN

•F2290004 — Merchant/Card acceptor- signature

•F2290005 — Other

•F2290006 — Merchant is suspicious

•F2290009 — Unknown; data unavailable.The default value.

Pin capture capability

Specifies whether the PIN is required to perform a transaction on the terminal

Cardholder activated terminal level

Terminal capability or cardholder's activity type determining transactions that can be performed on the terminal:

•F22D0000 — Transaction is not CAT transaction. This means that transactions that can be performed on the terminal are not of a cardholder activated terminal level type.

•F22D0001 — Automated dispensing machine with PIN capture possibility

•F22D0002 — Self-service terminal

•F22D0003 — Terminal with payment amount restriction

•F22D0004 — Purchasing of good in flight

•F22D0006 — Electronic commerce terminal

•F22D0007 — Transponder transactions

•F22D0009 — Mobile terminal

Available networks

Group that contains a list of networks whose cards can be processed

Available currencies

Group that contains a list of the currencies that are available for transactions

Payment possibility

Specifies whether payments can be made on the terminal

Cash in present

Specifies whether the terminal can accept cash

PIN block encoding format

The format used to convert PIN blocks when loading files with data on terminals to SVFE:

•PNBFANSI — ANSI

•PNBFDOCL — Docutel

•PNBFISO1 — ISO 9564-1 format1

Installment support indicator

Determines whether the POS terminal supports installment payment processing.

4.Click Save.

5.If the terminal is an ATM, additional ATM-specific parameters need to be configured in the ATM and Dispensers tab in the details pane.
