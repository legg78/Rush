Communication interfaces

Communication standards (interfaces) specify how SmartVista communicates with external systems at the application level (in terms of the OSI Model). Communications standards dictate such things as message formats and the rules governing their exchange. Within SmartVista, communication interfaces consist of sets of parameters that specify how SmartVista interacts with externals systems.

There are two categories of standards (interfaces):

•Authorization (used for online interaction)

•Clearing (used for offline, file-based interaction)

A communication device must be registered in the system as part of the host configuration and assigned to hosts for online transaction processing. Unlike communication standards, communication devices are applicable only to online communication and are responsible for the network and transport levels of interactions with external entities.

The communication device is assigned to the host, as configured in SmartVista, and provides interaction with the external host entity. All devices associated a host follow the online communication standard defined for that host, and inherit all the attributes specified in the online standard. However several device-specific parameters may be redefined per device.

The most recent version of a standard is used by the host interface and any devices defined with it. A starting date is assigned to each version of the standard. This is the date on which the version of the standard becomes effective.

Note: Parameter values defined at the interface without specifying a particular version are valid for all versions of the standard.

The version number and starting dates are configured at the host for both online and offline standards.

The parameters of communication standards are defined in the host interface. The values of these parameters can parameterize either a specific version of the standard or the standard itself, in which case the parameters are applicable to all versions.

Where many objects of a particular type (for example, ATMs) require parameters to be configured, manual configuration of each object one by one is impractical. Instead use is made of templates. Templates organize parameters common to a particular device type. Templates are thus intended for use with a group of objects (such as terminals) employing the same communication standard configuration. Assigning a template to a device applies all the standard’s settings to the device.

Note: The list of registered communication standards is provided on the Communication > Standards page.
