User management application wizard

This topic describes the user management application wizard forms as related to different application flows and wizard blocks.

Create user

This flow is selected to register a new user in the system.

 

The initial set of parameters that can be configured for the user includes:

Parameter

Description

Login

User account used to log in to the system.

Role

User role that corresponds to the user’s job responsibilities. You must select one role. If the user has several roles, then you should specify them later.

Default institution

Institution to which the new user belongs.

Person section

Details of the person to be associated with the user.

 

Change user

This flow is selected to modify the settings of an existing user. The wizard structure is similar to that of the Create user flow except for the absence of the Role and Default institution fields.

Change user’s status

This flow is selected to set the activity status of an existing user. The status is specified according to the user status dictionary (USST).

Change user’s roles

This flow is selected to set roles assigned to an existing user. The wizard provides:

1. Selecting a user by login (steps 1–3 in the figure below)

2.Assigning required roles to the selected user (step 4 in the figure below).

 

Application wizard blocks

The user management application consists of the following standard sections:

•Application — This section includes common application parameters.

•User — Including:

oPerson — Contains parameters relating to the person associated with the user.

oUser’s institution — Contains the parameters of the financial institution to which the user belongs.

oUser’s role — Contains the parameters of the role assigned to the user.
