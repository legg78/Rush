Viewing acquirer transactions

1.Click Acquiring > Operations.

The search pane consists of the following tabs:

Tab

Description

Operation

Search through all transactions in the database.

Document

Search through operations with documents (for example primary and payment documents) associated with operations, transactions, payment orders.

Participant

Search through all operations using participant parameters.

Payment order

Search through operations with payment orders.

Tags

Operation tags. The list of tags used can be found on the Tags page (see Operational rule > Authorization > Tags).

2.Click the Operation tab and specify the search parameters.

Parameter

Description

Host date from

Host date till

Inclusive to and from dates when the transaction was performed.

Operation ID

The transaction ID (required).

Terminal number

The ID of the terminal on which the transaction was performed.

Merchant number

The merchant ID of the terminal on which the transaction was performed.

3.Click Search.

The search result pane displays the list of operations that match the search criteria. Icons indicating the reversal or canceled status appear alongside the transactions. Transactions are marked with the following icons:

— canceled transactions

— reversals

The details pane has the following tabs that contain specific details about the currently selected transaction.

Tab

Description

Details

General information about the operation.

Accounting

Account type, balance type, and credit/debit amounts. The data can be viewed in simplified mode (if the Simplified view check box was selected).

Payment order

Payment order information.

Trace

The operation processing steps (authorization scenario states) in accordance with the set trace level.

Tags

Operation tags. The list of tags used can be found on the Tags page (see Operational rule > Authorization > Tags).

Authorization

Information about the authorization.

Participants

Detailed information about institution, card type, account number, amount, currency, and other issuer and acquirer objects involved in this transaction.

Messages

System generated messages during transaction processing.

Disputes

Disputes are initiated (based on some data) to verify the validity of the original transaction.

Additional amounts

Transaction amounts.
