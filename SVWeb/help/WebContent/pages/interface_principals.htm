User interface basics

The user interface displays only the pages, controls, and data that are permitted by your user rights. Contact your system administrator to configure appropriate access rights for you.

New pages open with the search result pane empty. On some pages, leaving all search fields blank returns all records while on others search criteria must be specified in order to return results. Clicking the Clear all link resets all search parameters.

Note: The sequence and number of the displayed columns can be altered on the search result page. These settings can be saved for future use, provided the system administrator has enabled this functionality.

A search mask can be used when specify search parameters:

•For exact searches, type the whole name or value in the corresponding input field. For example, entering Head returns all objects with the names such as Head and head (searches are not case sensitive)

oAn asterisk may be used as a wildcard in searches to replace zero or more characters:

▪Head* returns all objects that start with Head, such as Head1, Head Office, Head2, head88, etc.

▪*head returns all objects that end with Head, such as Main Head, etc.

▪*Head*, returns all objects that contain head, such as Central Head Office, head_office_south, etc.

oSelecting a value from a list results in an exact search on that value.

When specifying a time period, the start date begins at midnight (00:00 midnight), and the end date finishes at 23:59. For example, if you wished to search for transactions for the full date of 25 February, you would enter both start and end dates as 25.02 and 25.02.

Search parameter values are not saved when switching between pages. You can sort the data displayed in the search results and details panes.

Mandatory parameters in configuration windows are marked with asterisks. Changes can be saved only when all values for such parameters have been provided. If mandatory fields are omitted or if data is entered incorrectly (for example, an incorrect format is used), a message is displayed in red below the affected field:

Changes to parameters (for example, the Name parameter) take effect immediately after clicking the Save button.

Note: Changes to user roles, permissions or role assignments take effect only when the affected users next log in.

When you save a new object, it displays at the top of the result list regardless of the search criteria.
