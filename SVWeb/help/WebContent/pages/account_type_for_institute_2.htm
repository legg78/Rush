Creating an account type

An account type is a specific category of account that defines the account’s purpose. It is linked to an institution and determines the necessary balances and activity type for the account.

To set up an account type:

1.Click Operational rules > Accounting > Account types.

2.Click Add.

3.In the New account type window, specify the parameter values for the account type.

Parameter

Description

Institution

Target institution (institution to which the account type will be linked).

Account type

Specified account type. If the required account type is missing, click the Add account type link. Add a new article and then click Save.

Number format

Number format for the account number. If the number is to be generated automatically by SmartVista, enter a number format. A number format is not required if the account number is automatically obtained from an external system or can be manually entered by an operator.

Number prefix

Value to use for the ACCOUNT_TYPE_PREFIX parameter (if it has been added as a component of the account number naming format).

Product type

Product type to associate with the account type.

Balance type

From the Balance type list, select a balance type for the account structure.
Additional balances can be added later via the Balances tab of the bottom tab pane.

The following additional parameters can be configured:

•Default update macros — macro is applied for accounting transactions related to the balance, for instance, on performing the account adjustment, if no other macro is specified explicitly for it

•Balance algorithm — algorithm for calculating the balance amount if the balance is not real, but virtual; i.e., calculated on-the-fly by some algorithm. The algorithms are created by system developers.

Currency

Balance currency. If the balance currency value is omitted, it is inherited from the account. The account currency is specified directly on creating the account (automatically or manually).

Impact on available balance

Balance impact:

•Neutral — balance does not affect the available balance

•Positive — balance increases the available balance funds

•Negative — balance reduces the available balance funds

Status

Initial status of the balance:

•Active — an external system has been informed of the balance funds

•Closed — balance is closed, the funds flow is forbidden

•First entry — information of the balance funds is uploaded to the external system as soon as the balance is created (once uploading is complete, the balance status is changed to Active

•Inactive — after the initial call, the balance status is changed to First entry, information of the balance funds is uploaded to the external system, and then the balance status is changed to Active

Rate type

Indicates the currency rate of the balance to the account (if currencies differ).

Number format

Naming rule for the balance. If the parameter value is omitted, the name is not displayed.

Number prefix

Value to use for the BALANCE_TYPE_PREFIX parameter (if it has been added as a component of the balance number naming format).

4.Click Save.

5.Define the account structure.

6.Define the entities to establish accounts of the created type.
