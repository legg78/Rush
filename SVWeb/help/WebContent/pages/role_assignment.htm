User roles and privilege restrictions

Users are able to perform tasks according to the privileges permitted by the roles (and subroles) assigned to them. Privileges cannot be assigned to users directly; they must be assigned through roles.

If any role has changed (privileges added or deleted), or the user status has been updated, then any higher-level roles to which it belongs will change accordingly. Changes take effect after the next login.

A single role must be assigned to a user during registration. Additional roles can be added after the registration process is finished.

The privileges within the roles assigned to a user can be limited by restrictions. Restriction options are linked to the corresponding privilege. If restrictions are not explicitly set, the user can perform all the activities granted by the privilege without any restrictions.

If the same privilege is granted to a user through different roles with different restrictions set for each role, only the restrictions that are present in all roles take effect. If any role applicable to a user grants the privilege without restriction, then the user will enjoy the privilege without restriction.

Note: Users cannot change their own access rights.

The institutions and the agents to which a user has access should be specified during the user registration process together with the roles.
