Specifying card types for products

1.Click Issuing > Configuration > Products.

2.Specify search parameters and click Search on the search pane.

3.Click the card product on the search result pane.

4.Click the Card types tab, and click Add.

5.In the New product card type window, specify the card issuing parameters.

Parameter

Description

Card type

Type of card to be added to the product.

BIN

Institution BIN which is to be used for the card type.

Seq number low

Seq number high

Minimum and maximum order number for reissuing cards of that type.

Method

Card personalization method.

This is specified for the issuer’s card personalization module and is not used by SVBO.

Number format

Previously configured rule for card number generation.

Index range

Range of the BIN.

PIN request, PIN mailer request

Specify the need for PIN generation and PIN mailer (envelope) printing.

Embossing request

Embossing requirement for inclusion in the output embossing file:

•No plastic — the file is not required. This value is used to generate PIN, print PIN mailer, and issue a virtual card

•Plastic embossing — the file is generated to emboss card, write information on magnetic stripes and an application on the chip

Blank type

Specifies the design or type blank plastic card that will be used.

Online status

The card status after it is created in the system. In accordance with the bank requirements, the possible values are the following:

•CSTS0012 - Card is not activated — the card will be activated on its delivery or after the call center reference

•CSTS0014 - Forced PIN change — the card will be activated after the first PIN change

•CSTS0017 - PIN activation — the card will be activated after the first PIN transaction

Personalization priority

The urgency of the personalization:

•PRSP0100 - Express

•PRSP0500 - Normal

Reissuing command

Automatic reissuing feature (reissue with the same or a new card number, etc.).

Reissuing start date rule

Reissuing expiration date rule

Rules used in calculating the reissued card start date and expiry date.

Reissuing card type

Card type used for the next automatic reissuing.

Card state

The state for the card instance after its creation. In accordance with the bank requirements, the possible values are the following:

•Issued (personalization is not required; for example for virtual cards)

•Personalization required (cards need to be personalized)

Application scheme

Payment network in which scheme is used.

Service

Product service used for cards of this type.

Reissuing product

Card product that must be applied when the card is reissued (for cards that must be reissued with a new product).

Reissuing bin

BIN that must be used for the reissued card (for cards that must be reissued with a new BIN the Reissue command should be Reissue with a new number).

6.Click Save.
