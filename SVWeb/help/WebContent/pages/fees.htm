Fees

Fees are a special way of calculating charges based on the set of predefined formulas. Fees are charged for services provided by the bank as defined in the applicable contract. Examples of fees include:

•Account maintenance fees

•Card issuance fees

The amount of the commission (fee) is calculated according to the parameters that were set on the corresponding service term configuration.

A fee has the following parameters:

Parameter

Description

Fee type

Defines the fee purpose and describes its scope.

Method of fee rate calculation.

Possible values:

•Fixed value

•Percentage value

•Minimum between percent and fixed value

•Maximum between percent and fixed value

•Sum of percent and fixed value

 

•Method of fee base calculation

Either the transaction(s) sum or the number of transactions can be used for a fee base calculation. If a fee with a limit then a limit counter is attached to the base. Thus the counter can influence on the fee threshold in which the base falls. Similar to a fee base, a limit counter and fee thresholds consist of the following parts: the amount and number of transactions. The fee threshold is defined according to the limit counter and fee base (for example, a transaction counter accumulated $1,000, and the sum of the current transaction is $200; then the system will select the threshold due to the $1,200 of transactions). Then, the system defines the fee base (for example, there are two tiers: if the transaction amount less than $1,000 then the fee is 2%, otherwise it is 1%).

The following methods of the fee base calculation are available:

▪Incoming amount of the current transaction. For the example above, the fee is calculated as 200 × 1%.

▪Lower threshold of the current transaction. For the example above, the fee is calculated as 1000 × 1%.

▪Tiered basis: a fee value is calculated due to the rates of all thresholds. For the example above (if the fee rate calculation is a percent), the fee is calculated as 1000 × 2% + 200 × 1%.

▪Difference between incoming amount and lower threshold (for the above example the fee size is calculated as 1000 × 1%)

•Currency

The parameter defines the fee rate currency. When setting fee calculation, the user specifies the result currency. If this currency differs from the fee currency, the fee is converted into the desired currency. A currency rate type should be specified for each fee type. This rate type is used to convert the fee amount if its currency and the required currency differ.

If a fee is not required (for a transaction that is free of charge), a zero value must be explicitly specified.
